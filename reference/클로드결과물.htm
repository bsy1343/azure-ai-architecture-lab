서비스 개요

본 문서는 월간 방문자 50만 명 규모의 이커머스 웹 서비스에 대한 Azure 기반 아키텍처 설계를 설명한다. 서비스는 일반 소비자(B2C)를 대상으로 하며, 피크 시간대에 트래픽이 집중되는 특성을 가진다.


핵심 비기능 요구사항

  • 피크 시간대 트래픽 집중에 대한 안정적인 대응 (오토스케일 기반)
  • 월 예산 500만원 이내에서 운영 가능한 비용 구조
  • 고가용성과 비용 효율을 동시에 만족하는 균형 잡힌 설계

보안 / 가용성 체크리스트

NSG 정책
Web Subnet은 인터넷에서 443 포트만 허용, App Subnet은 Web Subnet에서만 접근을 허용하는 최소권한 구조로 구성됨
충족
Multi-AZ 구성
현재 단일 리전·단일 가용영역 구성으로, Availability Zone 분산 배포가 적용되어 있지 않음
검토필요
DR 전략
현재 재해복구(DR) 전략이 별도로 구성되어 있지 않음
미충족

용어집

NSG (Network Security Group)
Subnet 또는 NIC 단위로 인바운드·아웃바운드 트래픽을 필터링하는 상태 저장 방화벽
VMSS (Virtual Machine Scale Sets)
동일한 구성의 VM을 여러 대 묶어 트래픽에 따라 자동으로 확장·축소하는 서비스
서버리스 (Serverless)
사용량에 따라 자동으로 확장·축소되며, 유휴 시 일시중지되어 비용을 절감하는 실행 모델
Health Probe
Load Balancer가 백엔드 인스턴스의 상태를 주기적으로 점검해, 비정상 인스턴스로의 트래픽 전달을 차단하는 기능
